# Platform Core — Methodology

Стандарт архитектуры бизнес-логики для приложений, построенных на Platform Core. Не документация конкретного продукта — применимо одинаково к GreenMarket, Taxi, Postamat, Voice Assistant и любому будущему приложению на этой платформе.

## Откуда взялось

2026-07-08: в рамках работы над документацией GreenMarket (см. `../kwork/greenmarket_docs/v3_2026-07-08/`) пришла серия документов ТЗ-023…ТЗ-029 — формально как продолжение пакета «подготовка Customer UI к будущему FSM Engine». По итогам разбора выяснилось, что эти семь документов **не специфичны для GreenMarket**: они описывают архитектурные границы, контракт Domain Service, модель предметных объектов и принципы построения бизнес-процессов — то есть правила, которые в равной мере применимы к любому продукту на Platform Core.

Поэтому вынесены из архива GreenMarket в отдельную папку на уровне репозитория. GreenMarket-специфичный документ о подготовке именно этого продукта к FSM Engine (ТЗ-022) остался в `../kwork/greenmarket_docs/v3_2026-07-08/22_tz022_podgotovka_k_fsm_engine.md` — он ссылается на принципы отсюда, но сам является продуктовым, а не платформенным документом.

Перенесено как есть, без переработки содержания — только перенумерация файлов и обновление заголовков под новый контекст.
